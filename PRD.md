# Product Requirements Document: Digital Architect Portfolio (v1.0)

## 1. Project Overview

Product Name: Digital Architect Portfolio
Vision: A high-end, editorial-inspired portfolio for a full-stack developer that balances technical precision with a bold, minimal "Cyber-Industrial" aesthetic.

## 2. Target Audience

- Technical recruiters and hiring managers at high-growth tech companies.
- Potential freelance clients looking for high-fidelity engineering and design.
- The global developer community (for the blog section).

## 3. Core Principles

- Editorial Minimalism: High-contrast layouts, large typography, and intentional whitespace.
- Technical Personality: Use of code snippets, mono-spaced font elements, and "system status" indicators.
- Streamlined Navigation: A clean, fixed top navigation for easy access to Projects, Blog, and About.

## 4. Design System (Kinetic Grid / Cyber-Industrial)

- Primary Font: Space Grotesk (Modern, geometric, architectural).
- Secondary Font: Inter (Highly readable for body text and code).
- Color Palette:
  - Background: Deep Charcoal (#131314)
  - Primary Accent: Electric Indigo (#c0c1ff)
  - Secondary Accent: Soft Coral/Orange (#ffb690)
  - Surface Tones: Slate/Navy variations for cards and sections.
- Roundness: Sharp, 90-degree edges to maintain a structural, "built" feel.

## 5. Screen Specifications

### 5.1 Home / Hero Section

- Hero Statement: Large, bold typography introducing "Rafi Muhammad, Full-Stack Developer."
- Visuals: Abstract geometric patterns or subtle "tooling stack" indicators.
- Featured Works: A curated grid of "Crafted Solutions" showing project thumbnails and core tech.
- Social/Contact: Quick access to "Send Message" and "Copy Email."

### 5.2 Projects Gallery

- Filtering: Ability to view "All," "Side Projects," or "Production Apps."
- Cards: Large-format cards featuring project imagery, titles, and descriptions.
- Tech Stack Tags: Consistent labeling for languages and frameworks (Astro).

### 5.3 Project Detail

- Overview: Project title, subtitle, problem statement, solution, and role.
- Technical Stack: A dedicated section with icons for the "Engineered With" stack.
- Achievements: Numbered list highlighting key metrics (e.g., "99.9% Uptime").
- Gallery: Supporting imagery and mockups (Desktop/Mobile).

### 5.4 Blog Feed (Writings)

- Hierarchy: Large, impactful headlines for articles.
- Metadata: Clear indicators for Category, Date, and Read Time.
- Navigation: Search bar and category filters (Development, Design, Life).

### 5.5 Blog Post Detail

- Focus: Distraction-free reading experience.
- Features: Key Takeaways box, formatted code blocks with syntax highlighting, and "Metadata Tags."
- Author Bio: A clean footer section with the author's profile and social links.

### 5.6 About Page

- Manifesto: "The Architect of Digital Structures" introductory copy.
- Technical Stack: A structured grid of technologies categorized by "Modular Units."
- Professional Timeline: A vertical execution log of career milestones.
- Philosophies: Core design/engineering principles (Structural Purity, Security by Design).

## 6. Functional Requirements

- Responsive Design: Seamless transition from desktop to mobile views.
- Code Highlighting: Support for various languages within blog posts.
- Interactive States: Subtle hover effects on buttons and cards following the Design System.

## 7. Success Metrics

- Average time spent on project detail pages.
- Newsletter subscription growth (via Blog Feed).
- Contact form inquiries/Hiring requests.
